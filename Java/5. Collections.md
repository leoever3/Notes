### Collections

* A **collection** is a group of similar objects, gathered togher in a Java object.
  * Array of String
* Most java collection objects are inside **java.util** package.
* We can say collection is a database which keeps data into the **heap memory**.

#### <u>Collection Types</u>

* **List**
  * An **ordered** collection of Java objects
  * Allows duplicate
  * We need to use **index** to access to its elements 
* **Set**
  * similar to list
  * **not allow duplicate**
* **Queue & Stack**
  * Queue: its elements are ordered by **first-in-first-out**	double end
  * Stack: its elements are ordered by **first-in-last-out**	single end
  * We use queue/stack when we intend to sort the data before processing or we intend to add/remove elements in a specific order.

  * Map is a **key-value collection**.

#### ArrayList

* ArrayList is a java object which contains a set of other objects.
* ArrayList is one of the **fastest** and **most efficient** collection used in java.
* ArrayList is a **resizable** object and it is very useful for situation we don't know the size of the list in advance.

```java
List<String> mylist = new ArrayList<>();
mylist.add("Hello");

List<Integer> list = Array.asList(1,7,5,8,3);
Collections.sort(list);
```

#### LinkedList

* A LinkedList is both '**queue**' and '**list**'.
* We can read/add/remove from **both beginning or end** of the Linkedlist.
* ArrayDequeue is better than Linkedlist.

### Set

* Set is useful to **remove duplicate** object entries.
* Adding elements and checking in both have constant time, i.i. O(1).
* **Hashset** stores its elements in a **hash table**. It uses the HashCode() method to retrieve
* **Treeset** stores its elements in a **sorted tree** structure.
* DFS & BFS
  * Depth first search
  * Breadth first search
